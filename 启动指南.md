# 报表数据处理平台启动指南

## 项目结构

- **前端**：基于 Vue 2 + Element UI，位于项目根目录
- **后端**：基于 Spring Boot，位于 `report-backend` 目录

## 前端启动方式

### 开发环境启动

1. **安装依赖**（首次运行时）
   ```bash
   npm install
   ```

2. **启动开发服务器**
   ```bash
   npm run serve
   ```

3. **访问地址**
   - 本地：http://localhost:8081/
   - 网络：http://10.59.105.98:8081/

### 生产环境构建

```bash
npm run build
```
构建产物将生成在 `dist` 目录，可部署到 Nginx 等静态资源服务器。

## 后端启动方式

### 构建步骤

1. **进入后端目录**
   ```bash
   cd report-backend
   ```

2. **构建项目**
   ```bash
   mvn clean package -DskipTests
   ```

3. **运行应用**
   ```bash
   java -jar target/report-backend-1.0.0.jar --spring.profiles.active=dev
   ```

### 或使用项目根目录的脚本

1. **构建项目**
   ```bash
   ./build.sh all
   ```

2. **启动项目**
   ```bash
   ./start.sh
   ```

## 快速启动步骤

1. **启动前端**
   ```bash
   npm run serve
   ```

2. **启动后端**
   ```bash
   # 确保已构建
   cd report-backend
   java -jar target/report-backend-1.0.0.jar --spring.profiles.active=dev
   ```

## 注意事项

1. **依赖安装**：首次运行时需要安装前端依赖
2. **构建顺序**：后端需要先构建才能运行
3. **端口占用**：确保 8081（前端）和 8080（后端）端口未被占用
4. **环境变量**：后端默认使用 dev 环境配置

## 技术栈

- **前端**：Vue 2.6 + Element UI + Vue Router + Vuex
- **后端**：Spring Boot 2.1.2 + MyBatis Plus + Quartz + MySQL/H2

## 项目启动状态

- ✅ 前端开发服务器已启动：http://localhost:8081/
- ⚠️ 后端需要构建后启动
