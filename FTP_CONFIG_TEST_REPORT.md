## 3. 发现的问题

### 3.1 严重问题 (P0)

| 编号 | 问题描述 | 影响范围 | 建议优先级 |
|------|----------|----------|------------|
| BUG-001 | 密码明文存储 | 安全性 | 高 |

**BUG-001 详细说明**:
- **问题描述**: FTP配置中的密码以明文形式存储在数据库中，且API接口返回时也直接返回明文密码
- **影响范围**: 所有FTP配置的密码字段
- **安全风险**: 
  - 数据库泄露将直接暴露FTP服务器凭证
  - API接口返回明文密码，可能被日志记录或网络抓包获取
- **建议修复方案**:
  1. 使用AES加密算法加密存储密码
  2. 配置加密密钥存储在环境变量或配置中心
  3. API返回时对密码字段进行脱敏处理（如显示为******）

### 3.2 一般问题 (P1)

| 编号 | 问题描述 | 影响范围 | 建议优先级 |
|------|----------|----------|------------|
| BUG-002 | 后端参数校验缺失 | 数据完整性 | 中 |

**BUG-002 详细说明**:
- **问题描述**: 后端未实现参数校验，缺少必填字段时返回500错误而非400错误
- **影响范围**: 所有POST/PUT接口
- **建议修复方案**:
  1. 在实体类上添加@NotBlank、@NotNull等校验注解
  2. 在Controller方法参数上添加@Valid注解
  3. 使用BindingResult处理校验错误

### 3.3 已修复问题

| 编号 | 问题描述 | 修复状态 |
|------|----------|----------|
| BUG-003 | StatusTag组件状态映射错误 | ✅ 已修复 |
| BUG-004 | Docker FTP服务器被动模式连接失败 | ✅ 已修复 |

**BUG-003 详细说明**:
- **问题描述**: StatusTag组件中status=0和status=1的颜色和文本映射错误
- **原始代码**:
  ```javascript
  '0': 'success',  // 错误：0应该是禁用（红色）
  '1': 'danger'    // 错误：1应该是启用（绿色）
  ```
- **修复后代码**:
  ```javascript
  '0': 'danger',   // 正确：0是禁用（红色）
  '1': 'success'   // 正确：1是启用（绿色）
  ```

**BUG-004 详细说明**:
- **问题描述**: Docker FTP服务器使用桥接网络模式，被动模式地址配置为127.0.0.1，导致客户端无法连接到被动模式端口
- **根本原因**: Docker容器IP为172.17.0.x，但pasv_address设置为127.0.0.1，客户端无法正确连接
- **修复方案**: 重新创建Docker FTP容器，使用主机网络模式(--network host)
- **修复命令**:
  ```bash
  docker run -d \
    --name ftp-server \
    --network host \
    -e FTP_USER=ftpuser \
    -e FTP_PASS=ftppass \
    -e PASV_ADDRESS=127.0.0.1 \
    -e PASV_MIN_PORT=21100 \
    -e PASV_MAX_PORT=21110 \
    fauria/vsftpd
  ```
- **验证结果**: FTP连接测试返回`data: true`，连接成功

## 9. 操作日志系统

### 9.1 系统概述

为满足审计需求，已实现完善的操作日志记录系统，所有用户操作都会被详细记录。

### 9.2 记录的操作类型

| 操作类型 | 说明 | 记录内容 |
|----------|------|----------|
| CREATE | 新增操作 | 操作描述、目标对象 |
| UPDATE | 修改操作 | 操作前后数据对比 |
| DELETE | 删除操作 | 被删除对象信息 |
| TEST | 测试操作 | 测试结果 |

### 9.3 日志字段说明

| 字段 | 说明 |
|------|------|
| module | 操作模块（如：FTP配置） |
| operationType | 操作类型 |
| operationDesc | 操作描述 |
| targetId | 目标对象ID |
| targetName | 目标对象名称 |
| beforeData | 操作前数据（JSON格式，密码脱敏） |
| afterData | 操作后数据（JSON格式，密码脱敏） |
| result | 操作结果（1-成功，0-失败） |
| errorMsg | 错误信息（失败时记录） |
| operatorIp | 操作者IP |
| operatorName | 操作者名称 |
| duration | 执行时长（毫秒） |
| createTime | 操作时间 |

### 9.4 安全特性

- **密码脱敏**: 所有日志中的密码字段自动替换为`******`
- **异步记录**: 日志记录采用异步方式，不影响业务性能
- **完整追溯**: 支持操作前后数据对比，便于问题排查

### 9.5 API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/operation/log/page | GET | 分页查询操作日志 |
| /api/operation/log/{id} | GET | 获取日志详情 |
