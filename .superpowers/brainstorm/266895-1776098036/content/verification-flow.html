Trigger 分区防重复验证流程

多实例场景下的防重复机制验证

验证架构

实例 A (端口 8082)

  • TriggerJob 执行者
  • 处理 osd_sales_trigger
  • 标记 TRIGGERING → 执行 → TRIGGERED

实例 B (端口 8083)

  • TriggerJob 等待者
  • 检测到 TRIGGERING 状态
  • 跳过执行

MySQL 数据库

  • QRTZ_SCHEDULER_STATE - 实例心跳
  • QRTZ_LOCKS - 分布式锁
  • trigger_partition_record - 分区状态
  • trigger_execution_log - 执行日志

验证步骤

1

启动两个实例

实例A(8082) + 实例B(8083) 同时注册到 Quartz 集群

2

验证集群注册

查询 QRTZ_SCHEDULER_STATE，确认两个实例都注册成功

3

检查分布式锁

查询 QRTZ_LOCKS，确认 TRIGGER_ACCESS 锁正常工作

4

验证分区记录

查询 trigger_partition_record，确认状态为 TRIGGERED 且只有一条

5

验证执行日志

查询 trigger_execution_log，确认同一分区只有一条 TRIGGERED 记录

关键验证命令

-- 1. 查看集群实例
SELECT * FROM QRTZ_SCHEDULER_STATE;

-- 2. 查看触发器执行
SELECT * FROM QRTZ_FIRED_TRIGGERS;

-- 3. 查看分区状态
SELECT * FROM trigger_partition_record;

-- 4. 查看执行日志
SELECT * FROM trigger_execution_log 
ORDER BY id DESC LIMIT 10;
