## 1. 项目背景

### 1.1 业务背景

本项目作为RPA抓取数据上传到FTP服务器和BI报表之间的中间处理环节，负责：

1. 监听FTP服务器指定文件夹
2. 读取并解析Excel文件
3. 进行数据处理和转换
4. 生成BI报表可用的数据表

### 1.2 项目定位

- **定位**：轻量化数据处理中间件
- **目标用户**：企业内部数据分析团队
- **开发团队**：1-2人
- **报表数量**：几十个

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 自动化 | 自动监听FTP、解析文件、处理数据 |
| 可配置 | 通过配置管理报表，无需硬编码 |
| 可追溯 | 历史数据版本化管理，支持回溯 |
| 轻量化 | 单体架构，部署简单，维护成本低 |

## 4. 核心模块设计

### 4.1 FTP扫描模块

**职责**：定时扫描FTP服务器指定目录，检测新文件

**核心功能**：

| 功能 | 说明 |
|------|------|
| 多FTP支持 | 支持配置多个FTP连接 |
| 路径配置 | 支持配置扫描路径 |
| 文件匹配 | 支持正则表达式匹配文件名 |
| 扫描间隔 | 支持配置扫描间隔（分钟） |
| 去重处理 | 已处理文件不重复处理 |
| 完整性检测 | 文件大小稳定后再处理 |

### 4.2 Excel解析模块

**职责**：读取Excel文件，根据映射规则转换为标准数据结构

**核心功能**：

| 功能 | 说明 |
|------|------|
| 格式支持 | 支持.xlsx和.xls格式 |
| Sheet配置 | 支持配置Sheet索引或名称 |
| 行号配置 | 支持配置表头行号、数据起始行号 |
| 列名映射 | 根据列名匹配字段，支持列顺序变化 |
| 类型转换 | 支持STRING、INTEGER、DECIMAL、DATE类型 |
| 空值处理 | 支持默认值、跳过、报错策略 |

### 4.3 数据处理模块

**职责**：执行数据清洗、关联、聚合等操作

**核心功能**：

| 功能 | 说明 |
|------|------|
| 数据清洗 | 去重、空值处理、格式标准化、数据校验 |
| 数据关联 | 支持配置多表关联规则 |
| 数据聚合 | 支持配置聚合维度和聚合函数 |
| 数据转换 | 支持配置计算字段 |

### 4.4 输出管理模块

**职责**：将处理后的数据输出到目标存储

**核心功能**：

| 功能 | 说明 |
|------|------|
| 数据库输出 | 支持INSERT/UPDATE策略，自动添加pt_dt分区字段 |
| 文件输出 | 支持输出Excel/CSV格式 |
| 多数据库兼容 | 支持MySQL和GaussDB方言切换 |

### 6.2 异常处理

| 节点 | 异常类型 | 处理方式 |
|------|----------|----------|
| 连接FTP | 连接失败 | 记录错误日志，标记任务失败 |
| 匹配文件 | 无匹配 | 跳过本次扫描 |
| 检查已处理 | 已存在 | 跳过该文件 |
| 解析Excel | 文件损坏 | 记录错误日志，标记任务失败 |
| 列名映射 | 列名不存在 | 使用默认值或跳过该行 |
| 数据清洗 | 数据无效 | 记录到错误日志 |
| 写入数据库 | 写入失败 | 回滚事务，记录错误日志 |

## 11. 部署方案

### 11.1 部署架构

| 环境 | 数据库 | 部署方式 |
|------|--------|----------|
| 开发环境 | MySQL | 本地运行 |
| 生产环境 | GaussDB | 服务器部署 |

### 11.2 部署步骤

1. 后端打包：`mvn clean package -Dmaven.test.skip=true`
2. 前端打包：`npm run build`
3. 上传JAR包和前端静态文件到服务器
4. 配置application-prod.yml
5. 启动后端服务：`java -jar report-backend.jar --spring.profiles.active=prod`
6. 配置Nginx代理前端静态文件和后端API
